## Специфика Paper API
- Используй Paper API (не Bukkit/Spigot) — он предоставляет современные улучшения.
- Для текста используй `net.kyori.adventure.text.Component` и `MiniMessage` формат.
- События: `@EventHandler(priority = EventPriority.NORMAL, ignoreCancelled = true)`.
- Для асинхронных задач используй `Bukkit.getAsyncScheduler()` (Paper 1.20.5+) или `Bukkit.getGlobalRegionScheduler()`.
- Для работы с чанками, телепортациями и загрузкой миров используй `PaperLib`.
- Для создания GUI используй inventoryframework
- Регистрируй команды через Brigadier:
  ```java
  import io.papermc.paper.command.brigadier.BasicCommand;
  import io.papermc.paper.command.brigadier.CommandSourceStack;
- Для команд используй paper-plugin.yml с указанием api-version.

Асинхронность и многопоточность
- Никогда не блокируй главный поток сервера.
- Все I/O операции (БД, HTTP, файлы) — в асинхронных потоках.
- Для взаимодействия с Bukkit API (изменение мира, сущностей) — переключайся на синхронный поток.
- Используй CompletableFuture для цепочек асинхронных операций.
- Обрабатывай InterruptedException корректно.

Безопасность
- Валидируй весь ввод от игроков (команды, чат, GUI).
- Проверяй права (hasPermission()) перед выполнением действий.
- Используй prepared statements — никогда не конкатенируй SQL.
- Не логируй чувствительные данные (пароли, токены).
